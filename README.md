# Алгебра · Геометрия · Вероятность и статистика · ОГЭ — десктоп-сборка

Образовательное веб-приложение для 7–9 класса, упакованное в настольное
приложение: Python-бэкенд (локальный сервер + окно) плюс HTML/CSS/JS. Собирается
в один `.exe`, которому не нужны ни Python, ни сторонние библиотеки у пользователя.

История решений и находок (что сверялось, что оказалось неверным, что
исправлено и почему) — в [`CHANGELOG.md`](./CHANGELOG.md). Здесь — только то,
что в проекте есть сейчас.

Внутри — **четыре раздела**, переключаемых вкладкой наверху:

| Раздел | Учебник-ориентир | Тем | Заданий |
|--------|------------------|-----|---------|
| 🧮 Алгебра | Макарычев Ю.Н. и др. (7–9) | 59 | 357 |
| 📐 Геометрия | Атанасян Л.С. и др. (7–9) | 51 | 451 |
| 🎲 Вероятность и статистика | Высоцкий И.Р., Ященко И.В. (7–9) | 29 | 172 |
| 🎯 Подготовка к ОГЭ | Спецификация КИМ ОГЭ (ФИПИ), №1–25 | 21 | 279 |

Итого 160 тем и 1259 заданий, из них 128 с генераторами (бесконечная отработка
со случайными числами), 519 задач в банках глав, 22 анимации теорем и 45
достижений. Раздел ОГЭ сверен против официального банка ФИПИ и четырёх
независимых полных экзаменационных вариантов (см. CHANGELOG).

## Возможности

- **4 предмета**, каждый с теорией, интерактивными чертежами (Canvas, ~70
  типов — от простой числовой прямой до метода масс и биквадратных функций)
  и практикой; прогресс, XP, серии дней и 45 достижений сохраняются в
  `localStorage`.
- **Поиск** (🔍 в шапке или Ctrl/Cmd+K) — по названиям тем, формулам и
  строчкам теории сразу по всем предметам, с живым обновлением при вводе.
- **Онбординг** — 4 шага при первом запуске, дальше не мешает.
- **Генератор контрольных** — собирает вариант по главам/сложности/типам
  заданий, печатает или сохраняет как PDF. Для ОГЭ отдельно: кнопка
  «№6–25» (без практического блока) и «Полный вариант №1–25» (с блоком
  №1–5, без повторов внутри блока).
- **Картинка в условии vs в решении.** Для тем №1–19 (первая часть экзамена)
  чертёж, если есть, показан сразу в условии — как в реальном КИМе. Для
  №20–25 (вторая часть) чертёж скрыт до ответа и появляется вместе с
  разбором — тоже как на настоящем экзамене, где картинку во второй части
  обычно не дают. Это же поведение — и в печатном варианте (чертёж уезжает
  на лист ответов).
- **Карта знаний** (🗺️ с главной) — какая тема на какую опирается: узлы —
  темы (цвет = прогресс), стрелки — «пойми это раньше». Для ОГЭ каждый номер
  экзамена помечен чипом 🔗, ведущим к теме курса, из которой он вырос —
  тап переносит сразу туда, даже в другой раздел.
- **Мастерская** — потренировать один навык без давления; **Экзамен** —
  зачёт по теме целиком; **Кабинет учителя** — назначить тему, посмотреть
  прогресс класса.
- **Работает офлайн** после однократного `fetch_offline_assets.py` (шрифты
  + опционально KaTeX); без него — на системном шрифте, тоже без ошибок.
- **Экспорт/импорт прогресса** файлом (Настройки) — перенос между
  устройствами без сервера.

## Структура проекта

```
algebra-app-desktop/
├── web/                     # весь код приложения (HTML, CSS, JS)
│   ├── index.html
│   ├── favicon.svg
│   ├── css/
│   │   ├── style.css        # оформление, светлая/тёмная тема
│   │   └── fonts.css        # локальные @font-face (генерируется fetch-скриптом, см. ниже)
│   ├── fonts/               # сюда fetch-скрипт кладёт .woff2 (для работы без интернета)
│   ├── vendor/katex/        # сюда fetch-скрипт кладёт KaTeX (опционально, для матформул)
│   └── js/
│       ├── config.js        # константы (версия, XP за типы заданий, параметры экзамена)
│       ├── data.js          # разделы algebra и stats: миры, главы, темы, задания, достижения
│       ├── data-geometry.js # раздел geometry (merge в GAME в конце файла)
│       ├── data-oge.js      # раздел oge (практикум по номерам ОГЭ; merge в GAME в конце файла)
│       ├── plots.js         # ВСЕ интерактивные чертежи на Canvas (~70 типов)
│       ├── theorems.js      # готовые Canvas-анимации для теорем
│       ├── tasks.js         # рендер и проверка заданий (общие для всех разделов)
│       ├── srs.js           # алгоритм интервального повторения (SM-2-подобный)
│       ├── math.js          # опциональный рендер матформул $…$ через KaTeX (см. «Формулы»)
│       ├── state.js         # прогресс, достижения, серия дней, localStorage
│       └── app.js           # экраны, навигация, поиск, онбординг, генератор контрольных
├── main.py                  # бэкенд: локальный сервер + окно приложения
├── build.py                 # сборщик в .exe с гибкими настройками
├── build.config.json        # настройки сборки
├── requirements.txt         # зависимости (нужны только для сборки/нативного окна)
├── validate_content.js      # проверка целостности контента (node, годится для CI)
├── tools/
│   ├── fetch_offline_assets.py  # скачивает шрифты и KaTeX для полностью офлайн-сборки
│   └── build_single_html.py     # собирает всё в один HTML-файл (см. ниже)
├── CHANGELOG.md              # история решений и находок
└── build/                   # сюда попадёт готовый .exe (создаётся при сборке)
```

Порядок подключения скриптов в `index.html` важен: `config → data → data-geometry →
data-oge → srs → plots → theorems → tasks → state → app`. Разделы geometry и oge
подмешиваются в единый глобальный объект `GAME` merge-блоком в конце своих файлов.

## Запуск из исходников

```bash
python main.py
```

Окно открывается по приоритету: нативное окно (если установлен `pywebview`) →
окно-приложение Chrome/Edge без рамок → обычная вкладка в браузере по умолчанию.
Сервер поднимается на фиксированном порту (48841, с запасными), чтобы сохранения
в `localStorage` не терялись между запусками — они привязаны к адресу страницы.

## Сборка в один .exe

```bash
pip install -r requirements.txt
python build.py
```

Готовый файл появится в `build/`. PyInstaller не кросс-компилирует: `.exe` для
Windows собирайте на Windows, бинарник для macOS — на macOS, для Linux — на Linux.

## Автономный HTML (один файл для просмотра/отправки)

```bash
python tools/build_single_html.py
```

Собирает всё приложение (весь CSS и JS инлайном) в один файл
`algebra-stats-preview.html` — его можно открыть двойным кликом или переслать
одним файлом. Шрифты в этом режиме подключаются с Google Fonts (нужен интернет
для фирменного вида — это превью, а не офлайн-сборка `.exe`).

## Проверка контента

Перед сборкой (и в CI) стоит прогонять валидатор — он загружает те же файлы
данных, что и браузер, и проверяет целостность:

```bash
node validate_content.js
```

Проверяет: уникальность id (темы, задания, главы, миры, достижения, задачи);
корректность каждого задания по его типу (`choice`/`proof` — индекс `correct` в
диапазоне вариантов; `input` — числовой `answer` и допуск `tol`; `error` — есть
помеченный `wrong`; `match`/`proofSteps` — непустые пары/шаги); прогон всех 128
генераторов по 200 раз с проверкой, что они не падают и не выдают `NaN`; что
`cond` достижений не падает и не ссылается на несуществующие темы. ERROR → код
возврата 1 (сборку можно останавливать), WARN/INFO → код 0.

Текущий статус: **0 ошибок**, 3 предупреждения (косметические — см.
«Известные мелочи» ниже), 6 справочных.

## Работа без интернета (шрифты и формулы)

Приложение самодостаточно, **кроме шрифтов**: по умолчанию `index.html` берёт их
не из сети, а из `web/fonts/` через `web/css/fonts.css`. Чтобы наполнить эти папки
(нужен компьютер с интернетом — один раз):

```bash
python tools/fetch_offline_assets.py
```

Скрипт скачает подмножества шрифтов Rubik и Russo One (кириллица + латиница) в
`web/fonts/`, сгенерирует под них `web/css/fonts.css`, а также — опционально —
положит KaTeX в `web/vendor/katex/`. После этого собранный `.exe` показывает
фирменную типографику даже на машине без интернета при первом запуске.

Если папки пустые (скрипт ещё не запускали) — приложение не ломается: оно просто
использует системный запасной шрифт, а матформулы остаются в текущем текстовом
виде.

### Формулы (KaTeX) — опционально

`web/js/math.js` автоматически отрисовывает через KaTeX всё, что в контенте
обёрнуто в `$…$` (строчные) или `$$…$$` (выносные) — как только KaTeX скачан в
`web/vendor/katex/`. Это включается «по требованию»: существующий контент, где
формулы записаны юникод-символами (`a⁶`, `x²`), не меняется и рендерится как
раньше. Чтобы получить настоящую матвёрстку в новой теме, достаточно писать,
например, `$\frac{a}{b}$` или `$\sqrt{x}$` в полях `theory`, `q`, `example` —
`math.js` подхватит их сам, без правок в `app.js`.

## Наполнение курса

Все четыре раздела наполнены полностью (теория, интерактивный чертёж и практика
в каждой теме). Контент лежит в обычных `web/js/data*.js`. Чтобы добавить тему —
допишите объект в массив `topics` соответствующего раздела; при отсутствии тем
глава автоматически скрывается из списка (виден дружелюбный экран «в разработке»).

### Известные мелочи (см. вывод `validate_content.js`)

- Поле `world` у тем geometry и oge ссылается на неремапнутый id мира и сейчас
  навигацией не используется. Косметика, но при следующем слиянии данных лучше
  привести к общему виду (`geo_w7` и т.п.).
- Глава `misc` в geometry — служебный раздел без номера экзамена, банка задач
  у неё нет намеренно.

## Как обновлять приложение

Меняйте файлы в `web/` (обычные HTML/CSS/JS), прогоняйте `node validate_content.js`
и перезапускайте `python main.py`. Пересборка `.exe` нужна только для передачи
новой версии пользователям. Значимые решения и находки — фиксируйте в
`CHANGELOG.md`, а не здесь: README должен оставаться справкой о текущем
состоянии, а не журналом.
