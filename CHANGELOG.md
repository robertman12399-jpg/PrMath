## Сверка с официальным банком ФИПИ

Контент раздела ОГЭ сверялся по частям с выгрузкой официального банка ФИПИ
(файл с полными условиями и решениями №1–25).

- **Блок 1 (№1–6).** Практический блок №1–5 описывал 7 сюжетов, реально
  актуальны 6 (Участки, Квартиры, Бумага, Дороги, Сложные дороги, Мобильный
  интернет) — «Шины», «Печки» и «Тарифы»-такси таких сюжетов не существует
  *(позже, при сверке с другим источником, часть этого вывода была
  пересмотрена — см. ниже «Сверка с тремя полными вариантами»)*.
- **Блок 3, №20.** Было ошибочно описано как «системы уравнений» — по факту
  это уравнения 3–4 степени, решаемые группировкой/разложением на множители.
  Тема переписана *(тоже позже пересмотрено — системы вернулись обратно)*.
- **Блок 4, №22.** Было «сократимая дробь с выколотой точкой» — этого типа
  на тот момент не нашлось. Реальный формат — функция с модулем |x|,
  распадающаяся на две параболы-ветви; все вершины и условие «ровно 2 точки»
  проверены символьно (sympy) *(и это тоже пересмотрено позже — дырка
  оказалась реальной и даже более частой, чем |x|)*.
- **№21.** Добавлен реальный подтип «движение по реке с разницей во времени»
  (приводит к квадратному уравнению), проверен через sympy.
- **№17.** Добавлены 4 реальных подтипа: площадь трапеции по формуле,
  диагональ квадрата, сумма углов равнобедренной трапеции, диагональ
  трапеции делит среднюю линию — геометрия проверена координатами.
- **№18.** Добавлено «диагональ ромба на клетчатой бумаге».
- **№19.** Добавлены 2 задания в формате «впиши номера верных утверждений
  без пробелов» с реальными геометрическими ловушками.
- **№23.** Добавлена конструкция «окружность через две вершины треугольника».
- **№24.** Добавлено доказательство через центральную симметрию
  параллелограмма.
- **№25.** Добавлена задача уровня источника (перпендикуляр к AO из вершины
  B, где O — центр описанной окружности); факт AD=AB²/AC проверен численно
  (несколько треугольников, sympy/numpy).
- №10, №12, №14, №15, №16, №9, №13, №7, №8, №11 — сверены полностью,
  содержание подтвердилось верным.

**Банки задач глав (`problemSet`).** 83+ задачи, написанные до получения
файла ФИПИ (по данным math100.ru), прогнаны против подтверждённых реальных
подтипов: `pb_oge_ch3_9` (была система уравнений — заменена на группировку

ловушки — одно заменено на настоящую ложную ловушку), `pb_oge_ch4`
(добавлена задача на |x|, почти не представленная в банке главы). Остальные
~80 задач подтверждены корректными без изменений.

## Визуальные улучшения (проход по всем 160 темам)

Проверено соответствие чертёж↔тема по всему проекту, не только ОГЭ:

- **Найдена настоящая ошибка**, не просто слабое место: «Геометрическая
  прогрессия» показывала график y=x³ (кубическая функция) — концептуально
  неверно, прогрессия про экспоненциальный рост. Арифметическая прогрессия
  показывалась непрерывной линией вместо дискретной последовательности.
- Добавлен новый тип чертежа `sequence` — рисует прогрессию как отдельные
  точки a₁, a₂, … с направляющей линией/кривой, ползунком «сколько членов
  показать» и живым счётчиком Sₙ; закрывает и тему «что такое прогрессия»,
  и тему «сумма первых n членов» (включая сходимость частичных сумм для
  бесконечной геометрической прогрессии).
- «Решение систем уравнений второй степени» показывало тождество
  x(7−x)=7x−x² — не про системы. Добавлен новый тип `systemQuad` — парабола
  + прямая с ползунками, виден переход между 0/1/2 точками пересечения.
- Геометрия: у темы «Построение треугольника по трём элементам» чертежа не
  было вообще — добавлен.
- Остальные ~155 тем сверены — чертёж специфичен и уместен, либо
  универсальный тип используется по назначению (например `circle` с
  `polygon:6` в теме «Правильные многоугольники» — специально
  спроектированный вписанный n-угольник со слайдером).

По прямому вопросу «прошёлся ли ты по всем 100 заданиям из каждого
варианта?» — честный ответ был «нет»: первый проход был сканированием по

ошибка того же рода:

  целиком заменена на «функцию с |x|». При построчном каталоге по 12
  экземплярам дырка встретилась ДАЖЕ ЧАЩЕ (7 раз), чем |x| (5 раз). Оба
  реальны. Тема переписана: дырка возвращена (проверено делением многочлена
  столбиком через sympy), |x| осталась.
- **№9**: добавлен реальный формат ответа — конкатенация корней без
  пробелов в порядке возрастания.
- **№23**: добавлена «окружность с центром на стороне, касается другой
  стороны» — встретилась во всех трёх вариантах.
- **№14**: добавлена обратная формула суммы прогрессии («маляры красят
  забор»).
- **№17**: добавлена «высота ромба из вершины тупого угла делит сторону» —
  геометрия проверена координатами (numpy).
- **№19**: добавлены ещё 2 набора ловушек.
- **№20**: добавлено чистое биквадратное через t=x².
- **№21**: добавлена ловушка «средняя скорость» (гармоническое среднее).

## Картинка в условии vs в решении (№20–25)

Замечание: во второй части настоящего экзамена (№20–25) картинку в условии
обычно не дают — ученик представляет/строит фигуру сам, сверяется с
картинкой при проверке решения. Исправлено на двух уровнях:

- **Тренировка.** Новый флаг `revealFigureOnAnswer` на задании: чертёж
  скрыт, пока не отвечено, появляется (с анимацией) вместе с разбором.
  Проставлен на все 41 задание с картинкой в темах №20–25. Остальные темы
  (№1–5, 7, 10, 11, 13, 14, 16, 17, 18) — картинка сразу в условии, как и в
  реальном экзамене.
- **Печатный вариант.** Та же логика: чертёж переехал с листа условий на
  лист ответов. Проверено на 6 независимых генерациях.

Заодно нашёлся и починился независимый баг: чертёж на заданиях типа
`choice`/`proof` вообще не показывался в тренировке ни при каких
обстоятельствах — только в печати.

## Карта знаний

Новый раздел (🗺️ с главной): граф тем с прогресс-раскраской узлов и стрелками
«эта тема опирается на ту». Не строил дерево на все 160 тем вручную —
непрерывность порядка глав/тем и так очевидна из самой раскладки (колонки —
классы, внутри — главы по порядку). Вместо этого прицельно закурировал
**49 содержательных, НЕОЧЕВИДНЫХ связей** (не «следующий параграф», а
«этот приём реально использует тот»):

- **Все 21 тема ОГЭ → тема курса, откуда она выросла** (32 связи). Например
  №20 (уравнения высокой степени) ссылается на «Квадратный трёхчлен» и
  «Системы уравнений второй степени»; №16 (окружность) — на «Центральные и
  вписанные углы» и «Вписанная/описанная окружность». Использовал знание
  соответствий, накопленное за 4 сверки с реальными вариантами.
- **~17 внутрипредметных связей** на действительно нетривиальных местах:
  метод интервалов опирается на разложение квадратного трёхчлена (а не
  наоборот, как можно подумать по соседним параграфам); синус/косинус
  любого угла — на синус/косинус острого угла из 8 класса; теорема
  косинусов — на теорему Пифагора; длина окружности — на правильные
  многоугольники (предел при росте числа сторон).

Для связей ВНУТРИ предмета — кривая линия со стрелкой между узлами. Для
связей МЕЖДУ предметами (почти все связи ОГЭ, плюс несколько в курсе,
например «Координаты вектора» опирается на алгебраический «График функции»)
— рисовать линию через весь экран было бы нечитаемо, поэтому вместо этого
на карточке появляется чип 🔗 с иконкой предмета-источника; тап уводит сразу
туда, переключая раздел.

Технически: HTML-узлы (обычные кнопки, не canvas — чтобы клик работал
надёжно на телефоне) поверх SVG со стрелками; раскладка по колонкам
класс/глава вычисляется из уже существующего порядка тем в данных, без
отдельного алгоритма позиционирования графа. Проверено на всех 4 предметах:
количество узлов/рёбер/чипов на экране совпало с заложенным в данные
(59/6/0 — алгебра, 50/7/1 — геометрия, 29/2/1 — статистика, 21/0/32 — ОГЭ),
переход по клику (и по узлу, и по кросс-предметному чипу) проверен, 0 ошибок.
## Карта знаний — редизайн под эталонные mind-map

Первая версия карты была сеткой (колонки-классы, темы столбиком) — узлы
квадратные, тесные, подписи глав наслаивались, а смысловые связи шли поперёк
и пересекались. Переделал по канонам настоящих интеллект-карт (свёрка с
рекомендациями Buzan-стиля: радиально-древовидная структура от корня,
кривые органичные ветви, цветовое кодирование, скруглённые узлы, щедрый
white space, без пересечений):

- **Древовидная раскладка** корень(предмет) → класс → глава → тема, ветвится
  слева направо. Раскладка снизу вверх по числу листьев: каждой теме — своя
  строка, родитель центрируется по детям. Пересечений ветвей нет by
  construction (строго возрастающий X по уровням) — проверено: 0
  накладывающихся узлов на всех 4 картах (было главной проблемой).
- **Скруглённые узлы**: корень — крупный круглый бейдж с иконкой предмета на
  градиенте; классы — капсулы с цветной точкой; главы — мягкие карточки с
  цветной левой кромкой; темы — полностью круглые «таблетки»
  (`border-radius: 999px`), цвет обводки/фона по прогрессу.
- **Цвет ветви = класс** (синий/оранжевый/зелёный для 7/8/9), кривые Безье,
